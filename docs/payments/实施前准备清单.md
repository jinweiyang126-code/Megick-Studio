# 用户支付换积分 — 实施前准备清单

> 配套文档：[用户支付换积分改造方案.md](./用户支付换积分改造方案.md) · [国内个人出海收款选型.md](./国内个人出海收款选型.md)

在按 Phase 1 开始写代码之前，建议提前准备以下内容。

**若你在国内、无执照、用户主要在海外**，请优先阅读 [国内个人出海收款选型.md](./国内个人出海收款选型.md)，本清单中支付宝/微信相关项可跳过。

---

## 一、账号与资质（必做）

### 1. Stripe（全球主渠道）

| 准备项       | 说明                                                                                 |
| ------------ | ------------------------------------------------------------------------------------ |
| Stripe 账号  | [stripe.com](https://stripe.com) 注册，完成企业/个人认证                              |
| 可收款地区   | 确认主体所在国是否支持 Stripe Connect / 直接收款                                     |
| API 密钥     | Dashboard 获取`pk_` / `sk_`（测试环境先用 test key）                             |
| Webhook 端点 | 生产需公网 HTTPS 地址，如`https://api.yourdomain.com/api/webhooks/payments/stripe` |
| 收款币种     | 开通 USD、EUR、GBP、JPY 等（与套餐定价一致）                                         |
| 提现银行账户 | 绑定可接收对应币种的银行账户                                                         |

### 2. 支付宝（CNY）

| 准备项             | 说明                                                                      |
| ------------------ | ------------------------------------------------------------------------- |
| 支付宝开放平台账号 | [open.alipay.com](https://open.alipay.com)                                 |
| 企业主体           | 需营业执照（个人难开电脑/手机网站支付）                                   |
| 应用创建           | 创建「网页/移动应用」，签约「电脑网站支付」「手机网站支付」               |
| 密钥对             | 应用私钥 + 支付宝公钥（RSA2）                                             |
| `APP_ID`         | 开放平台应用 ID                                                           |
| 异步通知 URL       | `https://api.yourdomain.com/api/webhooks/payments/alipay`（需备案域名） |
| 同步跳转 URL       | 支付完成跳回，如`https://yourdomain.com/dashboard/billing`              |


### 3. 微信支付（CNY）

| 准备项           | 说明                                                        |
| ---------------- | ----------------------------------------------------------- |
| 微信商户平台账号 | [pay.weixin.qq.com](https://pay.weixin.qq.com)               |
| 企业主体         | 需营业执照 + 对公账户                                       |
| 商户号`mchId`  | 微信支付商户号                                              |
| 关联 AppID       | 公众号或开放平台应用 AppID（Native/H5 需要）                |
| API 证书         | APIv3 密钥、商户 API 证书、平台证书                         |
| 支付产品开通     | Native 支付（扫码）、H5 支付                                |
| 回调 URL         | `https://api.yourdomain.com/api/webhooks/payments/wechat` |

> **注意：** 支付宝和微信通常要求 **中国大陆备案域名**，海外服务器 + 未备案域名可能无法通过审核。

## 二、基础设施（必做）

| 准备项         | 说明                                                                  |
| -------------- | --------------------------------------------------------------------- |
| 公网 API 域名  | 如`api.megick.com`，必须 HTTPS                                      |
| Web 前端域名   | 如`megick.com`，用于支付完成跳转                                    |
| SSL 证书       | API 与 Web 均需有效证书（Let's Encrypt 即可）                         |
| Webhook 可达性 | 支付网关必须能访问你的 API；本地开发可用**ngrok / cloudflared** |
| Nginx 配置     | 生产环境注意`client_max_body_size`（项目 README 已有说明）          |
| 环境变量托管   | 生产密钥放部署环境，不要提交到 Git                                    |

Megick Studio 已有依赖，确认可用即可：

| 服务                   | 用途                                        |
| ---------------------- | ------------------------------------------- |
| MySQL 8                | 订单、套餐、流水                            |
| Redis 6+               | BullMQ 超时关单、查单补偿、邮件通知         |
| `APP_ENCRYPTION_KEY` | 加密支付密钥（`openssl rand -base64 32`） |
| `SESSION_SECRET`     | 已有，会话签名                              |
| SMTP                   | 充值成功邮件（可选，Phase 5）               |

## 三、业务与产品决策（开发前定好）

| 决策项     | 建议                                                   |
| ---------- | ------------------------------------------------------ |
| 套餐与定价 | 先定 3 档积分包 + 各币种价格（见改造方案第 11 节示例） |
| 默认币种   | 全球用户建议`USD`；国内可默认 `CNY`                |
| 赠送积分   | 是否做 bonus（如买 2000 送 200）                       |
| 订单超时   | 建议 30 分钟                                           |
| 退款策略   | 是否支持用户自助退款，还是仅管理员操作                 |
| 最低充值   | 是否设最低金额                                         |
| 发票       | 首期不做可明确写进文档                                 |

## 四、开发环境准备

```bash
# 1. 项目依赖（若尚未安装）
cd Megick-Studio
pnpm install

# 2. 环境文件
cp .env.example .env
cp apps/api/.env.example apps/api/.env.development.local
cp apps/web/.env.example apps/web/.env.development.local

# 3. 数据库与 Redis 本地可连
# DATABASE_URL、REDIS_HOST/PORT 配置正确

# 4. 本地 Webhook 隧道（开发支付回调时）
# ngrok http 3001
# 将生成的 https URL 填到 Stripe/支付宝/微信的测试回调地址
```

**支付 SDK（实施时再安装）：**

```bash
pnpm --filter @megick/api add stripe alipay-sdk wechatpay-node-v3
```

## 五、各渠道测试材料

### Stripe 测试

- 使用 **Test mode** 密钥
- 测试卡号：`4242 4242 4242 4242`（成功）
- Dashboard → Webhooks → 添加测试端点 → 获取 `whsec_` 开头的 webhook secret

### 支付宝沙箱

- 开放平台 → 沙箱环境
- 沙箱 `APP_ID`、沙箱密钥、沙箱买家账号
- 沙箱网关与正式环境不同，需单独配置

### 微信支付沙箱

- 商户平台可申请沙箱（部分产品支持）
- 或使用 **0.01 元** 小额真实支付做联调

## 六、合规与运营（上线前）

| 项                  | 说明                                                    |
| ------------------- | ------------------------------------------------------- |
| 用户协议 / 充值条款 | 说明积分不可提现、不退现金等                            |
| 隐私政策            | 支付信息由 Stripe/支付宝/微信处理                       |
| 积分规则            | 有效期、使用范围、退款规则                              |
| 客服渠道            | 支付异常、未到账处理流程                                |
| 日志与审计          | 订单、Webhook 事件需可追溯（方案中`PaymentEvent` 表） |

## 七、建议准备顺序

```
第 1 步（立刻可做）
  ├── 定好套餐与 USD/CNY 定价
  ├── 注册 Stripe 测试账号，拿到 test keys
  └── 本地 MySQL + Redis + 项目能跑起来

第 2 步（Stripe 上线前）
  ├── 公网 API 域名 + HTTPS
  ├── Stripe 生产账号认证 + 银行账户
  └── Webhook 端点配置

第 3 步（国内支付上线前）
  ├── 企业营业执照
  ├── 备案域名
  ├── 支付宝 / 微信商户入驻（周期通常 1–2 周）
  └── 签约对应支付产品

第 4 步（与开发并行）
  ├── 充值条款文案
  └── 管理后台操作人（谁管套餐、谁处理退款）
```

## 八、最小可启动清单（MVP）

若先做 **Stripe + 全球用户**，最少需要：

1. Stripe 测试账号 + test API keys
2. 本地或测试环境 API 可访问（ngrok 即可）
3. MySQL / Redis 可用
4. 3 档积分套餐 + USD 定价表
5. `APP_ENCRYPTION_KEY` 已配置

支付宝和微信可以 **Phase 3/4 再申请**，不阻塞 Phase 1–2 开发。

## 九、环境变量速查

实施时需在 `.env` / 管理后台配置的支付相关变量：

```bash
# Stripe (global)
STRIPE_SECRET_KEY=
STRIPE_PUBLISHABLE_KEY=
STRIPE_WEBHOOK_SECRET=

# Alipay (CNY)
ALIPAY_APP_ID=
ALIPAY_PRIVATE_KEY=
ALIPAY_PUBLIC_KEY=
ALIPAY_NOTIFY_URL=https://api.example.com/api/webhooks/payments/alipay
ALIPAY_RETURN_URL=https://example.com/dashboard/billing

# WeChat Pay (CNY)
WECHAT_PAY_MCH_ID=
WECHAT_PAY_APP_ID=
WECHAT_PAY_API_V3_KEY=
WECHAT_PAY_SERIAL_NO=
WECHAT_PAY_PRIVATE_KEY=
WECHAT_PAY_NOTIFY_URL=https://api.example.com/api/webhooks/payments/wechat

# Payments general
PAYMENTS_DEFAULT_CURRENCY=USD
PAYMENTS_ORDER_EXPIRE_MINUTES=30
```

完整说明见 [用户支付换积分改造方案.md](./用户支付换积分改造方案.md) 第 10 节。
