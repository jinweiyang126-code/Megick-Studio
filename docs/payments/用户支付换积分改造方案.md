# Megick Studio — 用户支付换积分改造方案

> 版本：v1.0
> 日期：2026-07-01
> 支付渠道：支付宝 / 微信支付 / Stripe
> 目标地区：全球

---

## 1. 背景与目标

Megick Studio 开源版当前仅支持**管理员在后台手动调整用户积分**，站点设置页明确提示：

> Open-source edition does not include online credit purchase flows. User credits are granted manually from the admin user detail page.

本方案目标是在开源版基础上，新增**用户在线支付购买积分**能力，支持全球用户通过 Stripe 支付，以及中国大陆用户通过支付宝、微信支付。

### 1.1 设计目标

- 支付确认后才发放积分，保证资金安全
- Webhook 幂等处理，防止重复入账
- 统一订单模型，多支付渠道可扩展
- 复用现有 `CreditsService` 与 `CreditLedger`，不破坏生成任务扣费逻辑
- 全球多币种定价，按地区/币种路由支付渠道

### 1.2 非目标（首期不做）

- 订阅制 / 会员月费
- 积分过期策略
- 小程序内支付（微信 JSAPI 可二期扩展）
- 发票开具

## 3. 总体架构

```
┌─────────────┐     ┌──────────────────────────────────────┐
│  用户 Web    │────▶│  payments 模块                        │
│ /dashboard/  │     │  OrdersService / FulfillmentService   │
│  billing     │     │  ProviderRouter                       │
└─────────────┘     └──────────┬───────────────────────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
        ┌──────────┐    ┌──────────┐    ┌──────────┐
        │  Stripe  │    │  支付宝   │    │ 微信支付  │
        │ (全球)    │    │ (CNY)    │    │ (CNY)    │
        └────┬─────┘    └────┬─────┘    └────┬─────┘
             │               │               │
             └───────────────┼───────────────┘
                             ▼ Webhook
                    ┌─────────────────┐
                    │ FulfillmentService│
                    │ (幂等入账)        │
                    └────────┬────────┘
                             ▼
                    ┌─────────────────┐
                    │ CreditsService   │
                    │ grantMonthlyAllowance │
                    └────────┬────────┘
                             ▼
                    ┌─────────────────┐
                    │ CreditLedger     │
                    │ refType=PAYMENT_ORDER │
                    └─────────────────┘
```

### 3.1 核心原则

1. **一种订单模型，三种支付实现** — 通过 Provider 适配器模式扩展
2. **支付成功后才发积分** — 订单状态：`PENDING → PAID → FULFILLED`
3. **Webhook 幂等** — `idempotencyKey = payment:{orderNo}`
4. **全球多币种** — 套餐按币种分别定价，不按单一汇率实时换算
5. **入账统一走 CreditsService** — 不直接修改 `Profile.credits`

### 3.2 支付时序

```
用户 → 选择套餐/币种/支付方式
     → POST /api/payments/orders（创建 PENDING 订单）
     → 跳转支付页 / 展示二维码（Stripe / 支付宝 / 微信）
支付网关 → POST /api/webhooks/payments/:provider
         → 验签 → 订单状态 PAID
         → FulfillmentService 幂等入账
         → 订单状态 FULFILLED
用户前端 → 轮询 GET /api/payments/orders/:orderNo
         → 显示到账积分
```

## 4. 数据模型

### 4.1 枚举定义

```prisma
enum PaymentProvider {
  STRIPE
  ALIPAY
  WECHAT_PAY
}

enum PaymentOrderStatus {
  PENDING      // 已创建，待支付
  PAID         // 支付成功，待入账
  FULFILLED    // 已发积分
  FAILED       // 支付失败
  EXPIRED      // 超时关闭
  REFUNDED     // 已退款
  REFUNDING    // 退款中
}

enum PaymentMethod {
  STRIPE_CHECKOUT    // 信用卡 / Apple Pay / Google Pay
  ALIPAY_PAGE        // 电脑网站支付
  ALIPAY_WAP         // 手机网站支付
  WECHAT_NATIVE      // 扫码支付
  WECHAT_H5          // 微信 H5 支付
  WECHAT_JSAPI       // 公众号/小程序（二期）
}
```

## 5. 全球渠道路由策略

### 5.1 路由规则

| 场景                        | 推荐渠道      | 支付方式                                  |
| --------------------------- | ------------- | ----------------------------------------- |
| 币种`CNY` + 桌面 Web      | 支付宝        | `ALIPAY_PAGE`                           |
| 币种`CNY` + 移动 Web      | 支付宝 / 微信 | `ALIPAY_WAP` / `WECHAT_H5`            |
| 币种`CNY` + 桌面（可选）  | 微信          | `WECHAT_NATIVE`（二维码）               |
| 币种`USD/EUR/GBP/JPY/...` | Stripe        | `STRIPE_CHECKOUT`                       |
| 全球信用卡用户              | Stripe        | Visa/Mastercard/Amex/Apple Pay/Google Pay |

### 5.2 币种选择逻辑

1. 首次进入充值页：按 `Profile.locale` + IP 地区（`CF-IPCountry` 或 `Accept-Language`）推断默认币种
2. 用户可手动切换币种（仅展示有定价的币种）
3. 切换币种后，可用支付方式随之变化

### 5.3 ProviderRouter 接口

```typescript
interface PaymentProviderOption {
  provider: PaymentProvider;
  paymentMethods: PaymentMethod[];
  displayName: string;
}

resolveProviders(input: {
  currency: string;
  locale: string;
  userAgent: string;
  countryCode?: string;
}): PaymentProviderOption[]
```

示例：

- `CNY` → `[{ provider: ALIPAY, ... }, { provider: WECHAT_PAY, ... }]`
- `USD` → `[{ provider: STRIPE, paymentMethods: [STRIPE_CHECKOUT] }]`

### 5.4 建议首期支持币种

| 币种 | 支付渠道     | 说明                            |
| ---- | ------------ | ------------------------------- |
| CNY  | 支付宝、微信 | 中国大陆及华人用户              |
| USD  | Stripe       | 全球基准定价                    |
| EUR  | Stripe       | 欧洲用户                        |
| GBP  | Stripe       | 英国用户                        |
| JPY  | Stripe       | 日本用户                        |
| HKD  | Stripe       | 香港用户（也可后续加支付宝 HK） |
| TWD  | Stripe       | 台湾用户                        |

## 6. 三渠道集成要点

### 6.1 Stripe（全球主渠道）

| 项       | 方案                                                                                |
| -------- | ----------------------------------------------------------------------------------- |
| 产品形态 | [Stripe Checkout](https://stripe.com/docs/checkout)                                  |
| 创建订单 | `stripe.checkout.sessions.create`                                                 |
| 回调路由 | `POST /api/webhooks/payments/stripe`                                              |
| 验签     | `stripe.webhooks.constructEvent(rawBody, sig, webhookSecret)`                     |
| 成功事件 | `checkout.session.completed`                                                      |
| 退款事件 | `charge.refunded` → 标记 `REFUNDED`，记负向流水                                |
| 配置项   | `secretKey`, `publishableKey`, `webhookSecret`, `successUrl`, `cancelUrl` |

**注意：** Stripe Webhook 需要 **raw body**，NestJS 需为该路由单独保留原始请求体，不能用全局 JSON parser。

### 6.2 支付宝

| 项       | 方案                                                                            |
| -------- | ------------------------------------------------------------------------------- |
| 产品形态 | 电脑网站支付 + 手机网站支付                                                     |
| SDK      | `alipay-sdk` 或官方 OpenAPI                                                   |
| 创建订单 | `alipay.trade.page.pay` / `alipay.trade.wap.pay`                            |
| 回调路由 | `POST /api/webhooks/payments/alipay`（`application/x-www-form-urlencoded`） |
| 验签     | RSA2 公钥验签                                                                   |
| 主动查单 | `alipay.trade.query`（Webhook 丢失时补偿）                                    |
| 配置项   | `appId`, `privateKey`, `alipayPublicKey`, `notifyUrl`, `returnUrl`    |

### 6.3 微信支付

| 项       | 方案                                                                            |
| -------- | ------------------------------------------------------------------------------- |
| 产品形态 | Native（扫码）+ H5                                                              |
| SDK      | `wechatpay-node-v3`                                                           |
| 创建订单 | `/v3/pay/transactions/native` 或 `/h5`                                      |
| 回调路由 | `POST /api/webhooks/payments/wechat`                                          |
| 验签     | 平台证书 +`Wechatpay-Signature` 头                                            |
| 主动查单 | `/v3/pay/transactions/out-trade-no/{orderNo}`                                 |
| 配置项   | `mchId`, `appId`, `apiV3Key`, `serialNo`, `privateKey`, `notifyUrl` |

### 7.4 创建订单

**请求：**

```json
{
  "packageCode": "starter-500",
  "currency": "USD",
  "provider": "STRIPE",
  "paymentMethod": "STRIPE_CHECKOUT",
  "returnUrl": "https://example.com/dashboard/billing/MG20260701001"
}
```

- `provider` 可选，不传则由 `ProviderRouter` 自动选择
- `returnUrl` 支付完成后跳回地址

**响应：**

```json
{
  "orderNo": "MG20260701001",
  "status": "PENDING",
  "credits": 500,
  "amountCents": 499,
  "currency": "USD",
  "expiresAt": "2026-07-01T14:30:00.000Z",
  "checkout": {
    "type": "redirect",
    "url": "https://checkout.stripe.com/...",
    "sessionId": "cs_xxx",
    "publishableKey": "pk_xxx"
  }
}
```

`checkout.type` 取值：

| type               | 说明              | 用于                         |
| ------------------ | ----------------- | ---------------------------- |
| `redirect`       | 跳转 URL          | Stripe / 支付宝 / 微信 H5    |
| `qrcode`         | 二维码内容        | 微信 Native                  |
| `stripe_session` | Stripe Session ID | Stripe Elements 嵌入（可选） |

### 7.5 入账逻辑

```typescript
// fulfillment.service.ts
async fulfillPaidOrder(orderId: string) {
  const order = await this.prisma.paymentOrder.findUnique({ where: { id: orderId } });
  if (!order || order.status !== 'PAID') return;

  await this.credits.grantMonthlyAllowance(
    order.userId,
    order.credits,
    `Credit purchase: ${order.orderNo}`,
    {
      refType: 'PAYMENT_ORDER',
      refId: order.id,
      idempotencyKey: `payment:${order.orderNo}`,
    },
  );

  await this.prisma.paymentOrder.update({
    where: { id: orderId },
    data: { status: 'FULFILLED', fulfilledAt: new Date() },
  });
}
```

### 7.6 订单状态机

```
PENDING ──支付成功──▶ PAID ──入账成功──▶ FULFILLED
   │                    │
   │超时                │退款
   ▼                    ▼
 EXPIRED            REFUNDING ──▶ REFUNDED
   │
   │支付失败
   ▼
 FAILED
```

合法状态流转必须由服务端控制，Webhook 和查单补偿均通过统一 `OrdersService.transitionStatus()` 方法执行。

## 9. 安全与合规

| 项           | 要求                                                 |
| ------------ | ---------------------------------------------------- |
| Webhook 验签 | 三渠道各自实现，验签失败返回 4xx                     |
| 幂等入账     | `idempotencyKey` + 订单状态机双重保障              |
| 金额校验     | 回调金额必须与订单`amountCents` 一致               |
| 密钥存储     | `CryptoService` 加密存入 `configEnc`，不入库明文 |
| 订单号生成   | 服务端生成（`MG` + 日期 + 序号），禁止客户端指定   |
| 超时关单     | BullMQ 定时任务，`PENDING` 超时 → `EXPIRED`     |
| 查单补偿     | 每 5 分钟对未过期`PENDING` 订单主动查单            |
| 退款处理     | 调用各渠道退款 API，账本记负向`CreditLedger` 流水  |
| PCI 合规     | 卡号信息不经过自建服务器，全部走 Stripe Checkout     |
| 限流         | Webhook 路由加 IP 白名单或 rate limit（可选）        |

## 15. 风险与注意事项

1. **Webhook 必须公网可达** — 本地开发可用 ngrok；生产 `API_BASE_URL` 必须正确
2. **幂等是核心** — 支付网关会重复回调，必须依赖 `idempotencyKey` + 状态机
3. **金额用最小货币单位存储** — 避免浮点误差
4. **Stripe raw body** — Webhook 路由不能走全局 JSON body parser
5. **微信/支付宝仅支持 CNY** — 全球用户默认走 Stripe + USD
6. **开源版无支付代码** — 属于新增模块，需完整开发，非开关启用
