# Megick Studio — 开源版与商业版差异

> 版本：v1.0  
> 日期：2026-07-02  
> 基于开源仓库代码、迁移记录与产品文案整理

---

## 1. 一句话概括

**开源版**是可自托管的完整 AI 创作平台，核心生成、管理、积分能力齐全；**商业版**（[megick.com](https://megick.com) 托管 SaaS）在相同创作能力之上，额外提供**在线付费、自助解锁高级权限、推荐返利**等运营变现模块——这些在开源仓库中**被移除或未包含**。

上游开源仓库：[zeeklog/megick-studio](https://github.com/zeeklog/megick-studio)

---

## 2. 功能对比总表

| 维度 | 开源版（本仓库） | 商业版（megick.com） |
|------|------------------|----------------------|
| **部署方式** | 自托管（Web + API + Desktop） | 官方托管 SaaS |
| **AI 图片生成** | ✅ 文生图、参考图、编辑等 | ✅ |
| **AI 视频生成** | ✅（需站点开关 + 高级权限） | ✅ |
| **模板中心 / 媒体中心** | ✅ | ✅ |
| **MegickCut 视频剪辑** | ✅ | ✅ |
| **Electron 桌面端** | ✅ | ✅ |
| **管理后台** | ✅ 用户/角色/模型/存储/队列等 | ✅ |
| **积分扣费 / 流水** | ✅ 生成任务自动扣费、失败退款 | ✅ |
| **在线购买积分** | ❌ 无 | ✅ Stripe / 支付宝 / 微信等 |
| **积分套餐 / 支付订单** | ❌ 无数据表与 API | ✅ |
| **推荐返利系统** | ❌ 已从数据库移除 | ✅ 邀请码、奖励转账等 |
| **自助开通高级访问** | ❌ | ✅ 付费或申请解锁 |
| **高级访问授权方式** | 仅 `SUPER_ADMIN` 角色自动拥有 | 付费用户 / 订阅用户等 |

## 3. 开源版明确移除或弱化的模块

### 3.1 在线支付 / 充值

README 明确说明：

> Credits system: administrator-managed credit adjustments, **without a built-in online purchase flow**.

多处 UI 与文案写死「开源版仅管理员手动调分」：

| 位置 | 说明 |
|------|------|
| `apps/web/src/routes/admin.site-settings.tsx` | 站点设置页提示横幅 |
| `apps/web/src/routes/dashboard.overview.tsx` | 积分卡片文案 |
| `apps/web/src/routes/dashboard.profile.tsx` | `creditModeManual` |
| `apps/api/prisma/seed.ts` | seed 日志说明无在线购买数据 |

代码层面 `CreditsService.grantMonthlyAllowance` 仍保留 `refType: "PAYMENT_ORDER"` 与 `idempotencyKey` 等商业版痕迹，但**无支付模块、无订单表、无 Webhook**。

详见：[用户支付换积分改造方案.md](./用户支付换积分改造方案.md)

### 3.2 推荐返利（Referral）

迁移 `apps/api/prisma/migrations/20260625000000_remove_referrals/migration.sql` 删除了：

- 数据表：`referral_invitations`、`referral_reward_transfers`
- Profile 字段：`referralCode`、`referralRewardCredits`、`referralRewardTotalCredits`
- 站点配置：`referrals.rewardCredits`、`referrals.autoApproveTransferLimit`
- 导航菜单：`/dashboard/referrals`

商业版具备完整邀请返利与审核转账流程；开源版**完全移除**。

### 3.3 高级访问（Advanced Access）变现逻辑

开源版 `AdvancedAccessService` 实现极为简化：

```typescript
// apps/api/src/common/services/advanced-access.service.ts
// 仅 SUPER_ADMIN 角色 → hasAdvancedAccess = true
const roleRows = await this.prisma.userRole.findMany({
  where: {
    userId: { in: uniqueIds },
    role: { code: "SUPER_ADMIN" },
  },
  // ...
});
```

**普通用户无法通过付费自助解锁**高级访问；需管理员赋予 `SUPER_ADMIN` 角色，或自行二次开发授权逻辑。

高级访问影响的能力：

| 能力 | 无高级访问时 |
|------|--------------|
| `accessLevel = PAID` 的模型 | 锁定，无法使用 |
| 视频 AI 工作室 | 需高级权限 |
| 生成结果原图 / 去水印 | 带水印或隐藏原图 URL |
| 直接访问部分 OSS 生成资源 | 403 禁止 |

中文文案（`zh-CN.ts`）：

> 开源版使用管理员手动授权，不提供在线申请解锁。

其他语言版本（如 `zh-TW.ts`）仍保留「充值解锁」类旧文案，说明商业版与开源版文案未完全统一。

### 3.4 用户侧运营页面

开源版缺少（或已删除）：

| 路由 / 功能 | 状态 |
|-------------|------|
| `/dashboard/billing` 充值页 | ❌ 不存在 |
| `/dashboard/referrals` 推荐页 | ❌ 已删除 |
| 在线申请高级权限 | ❌ 无流程 |

## 4. 开源版保留的核心能力

以下与商业版在**创作与运维**侧基本对齐，适合私有化部署：

- **AI 生成**：100+ 模型接入框架（OpenAI / 阿里云等协议）
- **图片**：文生图、参考图、AI 编辑（擦除/替换/扩图/文本编辑）
- **视频**：文生视频、图生视频（受站点开关控制）
- **MegickCut**：浏览器内视频剪辑、时间线、字幕、导出
- **模板中心**：公开模板、分类、审核、发布
- **媒体中心**：生成物与用户上传统一管理
- **管理后台**：用户、角色、模型、Provider、模板、存储、队列、审计日志、站点设置
- **积分体系**：余额、流水、生成扣费、失败退款
- **管理员调账**：单个/批量调整积分，可选邮件通知
- **注册赠分**：`auth.defaultRegistrationCredits`（默认 80）
- **认证**：密码登录、OAuth、邮箱验证（可配置）
- **存储**：阿里云 OSS 默认；Cloudflare R2 可选
- **桌面端**：Electron 壳 + 远程更新分发
- **异步任务**：BullMQ Worker、生成任务对账

## 5. 运营 / 商业模式差异

| 项目 | 开源版 | 商业版 |
|------|--------|--------|
| 积分来源 | 注册赠送 + 管理员手动发放 | 上述 + 用户在线购买 |
| 高级权限 | 管理员赋 `SUPER_ADMIN` 或改模型配置 | 用户付费自助开通 |
| 用户增长 | 无推荐返利 | 邀请码返利体系 |
| 服务条款 | 强调「管理员管理、无在线购买」 | 含付费条款与在线购买说明 |
| 源码 | GitHub 开源，可二次开发 | 变现模块未完全开源 |

服务条款（`en.ts` §5）开源版表述：

> We offer Free Services and administrator-managed Advanced Services in the open-source edition.  
> Open-source credit balances are adjusted by administrators.  
> There is no built-in online credit purchase flow in this edition.

## 6. 代码层面的商业版预留痕迹

开源版虽移除商业模块，但代码中仍可见「曾经存在」的设计：

| 痕迹 | 位置 | 说明 |
|------|------|------|
| `PAYMENT_ORDER` refType | `credits.service.ts` | 支付订单入账 |
| `idempotencyKey` | `CreditLedger` 表 | Webhook 幂等 |
| `hasAdvancedAccess` 字段 | API / 前端多处 | 高级权限门禁 |
| `accessLevel: PAID` | `AIModel` 表 | 付费模型分级 |
| `requiresWatermark` | 生成输出媒体 | 水印策略 |
| `billing/history` 懒加载注释 | `AGENTS.md` | 商业版曾有账单页 |
| Referral 表结构 | init 迁移 SQL | 后续迁移删除 |

这些痕迹说明：**二次开发支付模块时，入账与权限体系可以复用现有骨架**，无需从零设计。

## 8. 二次开发建议

若基于开源版补齐商业能力，建议优先级：

| 优先级 | 模块 | 文档 |
|--------|------|------|
| P0 | 在线支付换积分 | [用户支付换积分改造方案.md](./用户支付换积分改造方案.md) |
| P1 | 高级访问与支付联动 | 支付成功后自动 `grantAdvancedAccess`（需扩展 `AdvancedAccessService`） |
| P2 | 推荐返利 | 参考 init 迁移中已删除的表结构重新设计 |
| P3 | 订阅制 / 会员月费 | 当前开源版与非目标，需独立设计 |

国内个人、海外用户收款选型见：[国内个人出海收款选型.md](./国内个人出海收款选型.md)
