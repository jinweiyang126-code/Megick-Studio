# 国内个人开发者 — 海外用户收款选型

> 适用场景：**人在国内 · 无营业执照 · 用户主要在海外**  
> 配套文档：[用户支付换积分改造方案.md](./用户支付换积分改造方案.md) · [实施前准备清单.md](./实施前准备清单.md)

---

## 1. 结论摘要

| 渠道 | 是否推荐（本场景） | 原因 |
|------|-------------------|------|
| **Stripe** | ✅ 推荐（上线主力） | 海外用户信任度高、费率低，与 Megick 改造方案一致 |
| **Creem / Paddle（MoR）** | ✅ 可作过渡 | 无需执照即可收款，适合验证市场 |
| **支付宝 / 微信** | ❌ 暂不需要 | 需执照 + ICP 备案，且用户主要在海外 |

**推荐节奏：** 现在用 Stripe 测试环境开发 → 上线前办美国 LLC 开 Stripe Live（或短期用 MoR）→ 支付宝/微信等有执照且国内用户增多后再做。

---

## 2. 三条渠道对个人主体的支持

| 渠道 | 纯个人（无执照） | 个体工商户 | 有限公司 |
|------|------------------|------------|----------|
| **Stripe** | ❌ 中国大陆主体不可直接注册 | ✅（需海外主体） | ✅ |
| **支付宝网站支付** | ❌ | ✅ | ✅ |
| **微信支付商户号** | ❌ | ✅ | ✅ |

> Stripe [官方支持地区](https://docs.stripe.com/get-started/account) **不包含中国大陆**。国内个人不能直接用大陆身份注册 Stripe 收款。

## 3. 推荐路径

### 路径 A：美国 LLC + Stripe（长期推荐）

与 [用户支付换积分改造方案.md](./用户支付换积分改造方案.md) 中 Phase 2 **完全兼容**，长期最稳。

| 阶段 | 行动 | 成本 / 周期 |
|------|------|-------------|
| **现在** | 注册 [Stripe 测试账号](https://dashboard.stripe.com/register)，获取 `pk_test_` / `sk_test_` | 免费，即时 |
| **开发期** | 按 Phase 1–2 完成支付模块与充值页联调 | — |
| **上线前** | 注册美国 **Wyoming LLC** | Atlas 约 $500；Doola/Firstbase 约 $200–400；2–4 周 |
| **配套** | 申请 **EIN** → 开 **Mercury** 等美国银行账户 | EIN 自办免费；Mercury 免费 |
| **激活** | 用美国公司主体完成 Stripe Live 审核 | 1–3 天 |
| **提现** | Stripe → 美国银行 → PingPong/连连等结汇至国内卡 | 保留合同、发票、对账单 |

**优点**

- 费率约 **2.9% + $0.30**（低于 MoR）
- 与 Megick Studio 原生 Stripe Checkout 设计一致
- 海外 B2B/B2C 用户信任度高

**缺点**

- 首年成本约 **$200–700**
- 需处理美国公司年报、税务申报（可委托代理）

**常见办理方式**

1. **[Stripe Atlas](https://stripe.com/atlas)** — 一站式：LLC + EIN + Stripe + Mercury，约 $500 + $100/年
2. **Doola / Firstbase** — 仅注册 Wyoming LLC，自行申请 EIN 与银行

---

### 路径 B：MoR 代收款（无执照过渡方案）

由 **Paddle**、**Creem** 等平台作为 Merchant of Record（商户记录方）代收款项，处理全球 VAT/GST，净额结算给开发者。

| 平台 | 费率 | 国内个人 | 说明 |
|------|------|----------|------|
| **Creem** | 约 3.9% + $0.40 | 相对友好 | 较新，适合 indie / AI 工具 |
| **Paddle** | 约 5% + $0.50 | 需与客服确认 | 成熟，企业客户多 |
| **Lemon Squeezy** | 约 5% + $0.50 | ⚠️ 2026 起大陆新主体难申请 | 已被 Stripe 收购 |

**优点**

- 通常 **无需营业执照**
- 1–3 天可上线收款
- 全球税务由平台代缴

**缺点**

- 费率高于 Stripe 直连
- 需额外开发 **MoR Provider 适配器**（跳转 MoR 收银台 + Webhook 发积分）
- 与当前方案中的 Stripe Checkout 流程不同

**何时选 MoR**

- 想先验证付费意愿，暂不想花 $200+ 办公司
- 月收入 &lt; $5,000，税费合规希望外包给平台

---

### 路径 C：不推荐

| 方案 | 原因 |
|------|------|
| 支付宝 / 微信 API | 需执照 + ICP 备案；本场景用户主要在海外，投入产出比低 |
| 国内个人 PayPal | 不适合 SaaS 自动发积分；费率高、易风控 |
| 虚假材料注册 Stripe | 资金冻结风险极高 |

## 4. 实施时间线（本场景）

```
第 1 周（立刻，零成本）
├── 注册 Stripe 测试账号
├── 启动 Phase 1 开发（数据模型 + payments 模块）
├── 定 USD 三档套餐定价
└── 准备英文站：产品介绍、隐私政策、退款条款、联系方式

第 2–4 周（开发）
├── 完成 Phase 1 + Phase 2（Stripe 测试环境联调）
├── 本地 Webhook：ngrok http 3001
└── 暂不开发支付宝 / 微信（Phase 3–4 跳过）

上线前 1–2 个月
├── 选定收款路径：
│   ├── 长期运营 → 办 Wyoming LLC + Stripe Live（路径 A）
│   └── 先验证市场 → Creem / Paddle（路径 B），后续再迁 Stripe
├── 公网 HTTPS API + 域名
└── Stripe 审核材料：真实产品页、条款页、客服邮箱

有稳定收入后
├── 结汇走 PingPong / 连连等合规渠道
├── 保留服务合同、Stripe 对账单、发票
└── 咨询税务（个税、CFC 等，收入规模较大时）
```

## 5. 与 Megick Studio 开发阶段的对应

| 收款选择 | 开发范围 |
|----------|----------|
| **Stripe LLC（路径 A）** | 按改造方案 Phase 1 → 2 执行即可 |
| **MoR（路径 B）** | Phase 1 照做；支付层增加 MoR 适配器；Stripe 可后补 |
| **仅支付宝/微信** | 本场景不推荐，Phase 3–4 可整体延后 |

改造方案中 Phase 3（支付宝）、Phase 4（微信）对本场景 **标记为可选**，有执照且国内用户占比上升后再实施。

## 7. 常见问题

**Q：Stripe 测试账号需要海外主体吗？**  
不需要。测试模式注册即可用，无需审核。

**Q：海外用户能用支付宝付款吗？**  
不能作为本方案的主通道。Stripe 支持部分跨境 Alipay/WeChat Pay 收款方式，但需 Stripe 已激活的海外主体商户开通，且面向的是「用支付宝的海外华人」，不是国内 API 直连方案。

**Q：办美国公司麻烦吗？**  
Stripe Atlas 约 2–3 周全自动；自建 LLC 约 4–6 周但更便宜。许多国内独立开发者走这条路径。

**Q：MoR 和 Stripe 能同时接吗？**  
可以。改造方案 Provider 适配器模式支持多渠道并存；本场景可先做 Stripe，MoR 作备用。

**Q：收入如何合法结汇到国内？**  
常见路径：Stripe → 美国银行账户 → 跨境支付平台（PingPong/连连）→ 国内银行卡。保留服务贸易合同与对账单备查。
