# 模版中心打开性能分析

本文档记录首页点击「模版中心」进入 `/templates` 体感偏慢的排查结论与后续优化方向。与 [Dashboard 侧边栏切换性能分析](./Dashboard侧边栏切换性能分析.md) 中「全页导航 vs 客户端切页」的对比可交叉阅读。

---

## 1. 现象

- 在平台首页 Header 点击「模版中心」（或等价导航入口）进入公开模板页时，打开偏慢。
- Network 中模板 API（`/api/templates`、`/api/templates/categories`）往往只需约 200ms，但整页可交互明显更长。
- 批量导入模板后，首屏卡片更多、示例图/视频请求更多，体感可能进一步变差。

---


相关文件：

|------|------|
| 首页 Header 菜单 | `apps/web/src/components/Header.tsx`（`<a href>` 渲染菜单项） |

## 3. 根因分析（多层叠加）

### 3.1 全量文档导航（影响最大）

Header 中动态菜单使用普通锚点，而非 TanStack Router `Link`：

```tsx
// apps/web/src/components/Header.tsx（示意）
<a key={link.id} href={link.href}>...</a>
```

后果：

- **非整站客户端切页**：每次进入模板中心近似「重新进站」；
- 无法使用 `preload="intent"` 做悬停预加载；
- HTML、主包 / 路由 chunk、根布局第三方脚本会重新参与首屏时间线。

对比：Dashboard 侧边栏已尽量用带 preload 的 `Link`，切换成本明显更低。

### 3.2 SSR loader 阻塞文档返回

`/templates` 路由 loader 通过 `createServerFn` 在服务端并行请求：

- `GET /api/templates?compact=true&page=1&pageSize=20`
- `GET /api/templates/categories`

文档 HTML 返回前需等待上述请求与渲染完成。服务端到 API 的往返、冷启动、或列表侧偶发慢查询，都会直接拉长「点到出内容」的时间。

相关：曾出现 MySQL `Out of sort memory (1038)` 导致列表接口失败；已通过「先按 id 分页再加载行」与列表索引修复（见 `templates.service.ts` / migration `prompt_templates_public_list_idx`）。接口恢复正常后，文档仍可能被 loader 正常耗时拖住。

### 3.3 子路由曾接不住父 loader 数据（已改）

- loader 定义在父路由 `/templates`（`templates.tsx`）；
- 原先页面在 `/templates/` 内调用本页 `Route.useLoaderData()`，拿不到父数据 → 客户端再打 page=1。

**现状（2026-07-14）：** `templates.index.tsx` 使用 `getRouteApi("/templates").useLoaderData()`；`TemplateCenterPage` 对 loader 注入的 `initialData` 设置 `initialDataUpdatedAt`，配合 `staleTime: 300_000` 避免立刻重复请求。

**注意（2026-07-15 已修）：** loader 只拉**无筛选** page=1。`initialData` 不得灌入带 `q` / `category` / `type` 的 queryKey，否则 `staleTime` 内不会 refetch，表现为搜索/分类「完全无效」。

### 3.4 公开页视觉壳偏重

`templates.index.tsx` 使用 `SiteLayout`，其中固定挂载 `GlazeBackdrop`（`shaders/react` WebGL：`Swirl` / `ChromaFlow` / `FlutedGlass` / `FilmGrain`）。

从首页再进模板中心会再次初始化重背景与布局壳，比 Dashboard 内「壳不动、只换 Outlet」更沉。

### 3.5 首屏媒体与列表体积

`TemplateCenterPage`：

- 首屏 `pageSize` 约 20（`TEMPLATE_PAGE_SIZE`）；
- 卡片使用 `exampleUrl` / `referenceUrls`；
- 视频预览 `preload="metadata"`，图片 `loading="lazy"`。

导入大量模板后：

- 首屏可见卡片多，OSS 签名跳转 / 外链 / 错误 URL 会占连接；
- 视频 metadata 并发拉会加重 Time-to-Visual-Complete。

列表 API 使用 `compact=true` 可减少参考 URL 展开，**不能**去掉卡片封面加载。

### 3.6 静态资源 RTT 与第三方脚本

实测常见现象（本地浏览器 Network）：

| 请求 | 观察 |
|------|------|
| 文档 `templates` | 约数百 ms～1s 级 |
| JS chunk（即使 304） | 仍可能数百 ms（往返延迟） |
| `site.webmanifest` 等 | 偶发 >1s |
| `stats.zeeklog.com/script.js` | 403，占连接但不挡业务 |
| Google One Tap / GSI | origin 未配置时控制台报错，额外噪声 |

瓶颈常在 **RTT / 整页刷新次数**，而非单次业务 API 体积。

## 4. 「感觉慢」与阶段对应

| 阶段 | 用户感知 | 主要成本 |
|------|----------|----------|
| 点击 → 页面出壳 | 白屏 / 整页刷新 | `<a>` 全量导航 + SSR loader 等 API |
| 出壳 → 列表有数据 | 区域空 / Loading | chunk 下载与水合；可能客户端二次拉列表 |
| 列表有了 → 视觉齐 | 栅格半空、封面陆续出现 | 示例图/视频预览与失败重试 |

当 API～200ms 仍觉得慢时，重点不应只盯接口，而应看 **整页导航 + Glaze + 媒体并发**。

---

## 5. 与 Dashboard 模板入口的差异（参考）

| 维度 | 首页 `/templates` | Dashboard 内模板 |
|------|-------------------|------------------|
| 导航方式 | Header `<a href>` 整页刷新 | 侧栏 `Link`（可 preload） |
| 布局壳 | `SiteLayout` + GlazeBackdrop | DashboardShell（侧栏常驻） |
| 首屏数据 | 父 loader + 可能客户端补拉 | 多为客户端 query（壳已在） |

因此「首页进模版中心」通常比「已在 Dashboard 里点模板」更慢，符合架构差异。

## 6. 优化建议（按优先级）

### P0：导航改为客户端路由

- Header（及移动菜单）对站内路径使用 TanStack `Link`；
- 对 `/templates` 启用 `preload="intent"`（注意 Router preload 竞态，参见 Dashboard 文档）；
- 预期：去掉整页刷新与大量重复静态请求，体感提升最大。

### P0：保证首屏用上父 loader 数据

- 子页显式从父路由读取 loader（如 `getRouteApi("/templates").useLoaderData()` 或等价写法）；
- 避免 `initialTemplatesPage` 为空导致客户端再打一发首屏列表。

### P1：减轻公开模板页视觉成本

- 模板列表页延迟挂载 / 静态降级 `GlazeBackdrop`；
- 或模板路由改用更轻的 layout（保留 Header/Footer、去掉 WebGL）。

### P1：收敛首屏媒体

- 首屏视频不宜全部 `preload="metadata"`（改为可见再加载，或仅首屏少量）；
- 封面失败（404 / 跨域）时快速占位，避免长时间挂起；
- 继续保证列表走 `compact` + 分页。

### P2：基础设施

- 静态资源 / HTML 边缘加速，降低 304 校验 RTT；
- 梳理根布局中非关键第三方脚本，避免与首屏抢连接；
- 监控 `GET /api/templates` P95，防止回归到排序内存类故障。

---

## 7. 建议验收

1. 首页点击「模版中心」：**无整页 document 重载**（或仅软导航）；Network 中主文档不应每次都完整重拉业务 shell。  
2. 首屏列表：首包即可出格子（父 loader 数据生效），客户端不再立刻重复请求 page=1（除非 stale）。  
3. `GET /api/templates?compact=true&page=1&pageSize=20` P95 保持稳定（修复 1038 后不回归）。  
4. 弱网下首屏可见封面数量可控，不因 20 路视频 metadata 堵死。
